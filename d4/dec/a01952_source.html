93 /** \brief base class for 2D plotter classes (used by the plotter widget JKQTPlotter)
94  * \ingroup jkqtpplotterclasses
95  *
96  * This class implements basic functionalities for the plotter classes. Those are:
97  * -# data storage using a JKQTPDatastore object
98  * -# coordinate transforms
99  * -# a set of properties for the graphs (colors, widthes ...) and also a system (see getNextStyle() to automatically
100  * choose a drawing style for different graphs.
101  * -# plot a set of overlay elements (may be used for fast plotting of indicators onto a complex plot)
102  * -# drawing the coordinate axes, grids ... (logarithmic and linear)
103  * -# saveing and printing the resulting plots
104  * .
105  *
106  * This class is NOT a widget, if you need a plotting widget, use JKQTPlotter. This class may be used to
107  * plot using a JKQTPEnhancedPainter.
108  *
109  * \see JKQTPlotter a QWidget class that displays a JKQTBasePlotter plot on a Qt Window.
110  *
111  *
112  * \tableofcontents
113  *
114  *
115  * \section jkqtplotter_base_datastore Data Storage
116  * As already mentioned this class does not provide means to draw graphs, but it contains a basic mechanism to associate
117  * data with it. This is imeplemented, using a JKQTPDatastore object together with some convenience access methods. No further
118  * data access is implemented and child classes are NOT forced to use this datastore. Hide it if you want. JKQTPlotter shows
119  * how to use it. This class implement a graph management, where graphs simply point to a set of columns inside the datastore
120  * that contain the actual plot data!
121  *
122  * If you call the \link JKQTBasePlotter::JKQTBasePlotter() constructor \endlink with no arguments, it will create an internal
123  * datastore object and you can start adding data by using getDatastore(). If you have an external JKQTPDatastore object you can
124  * give it as parameter to the constructor or use one of the other methods:
125  * - useExternalDatastore(): \copybrief JKQTBasePlotter::useExternalDatastore()
126  * - useAsInternalDatastore(): \copybrief JKQTBasePlotter::useAsInternalDatastore()
127  * - useInternalDatastore(): \copybrief JKQTBasePlotter::useInternalDatastore()
128  * - forceInternalDatastore(): \copybrief JKQTBasePlotter::forceInternalDatastore()
129  *.
130  *
131  *
132  *
133  * \section jkqtplotter_base_systems_baseplotter Coordinate Systems and Transformations
134  * These topics are discussed in the help for JKQTPCoordinateAxis. There is a set of coordinate transform
135  * methods (x2p(), y2p(), p2x(), p2y() which only call the respective methods in xAxis and yAxis objects.
136  *
137  *
138  * \section jkqtplotter_base_grids_baseplotter Axis Ticks and Grids
139  * - The plotting of coordinate axes and grids, as well as coordinate transforms is done by
140  * JKQTPCoordinateAxis descendents (see documentation there)
141  * .
142  * If you want to set the axis properties, use getXAxis() or getYAxis() to get a pointer to the axis objects which then
143  * may be used to set the axis properties.
144  *
145  *
146  * \section jkqtbaseplotter_appearance_and_style Appearance & Styling of the Graph
147  *
148  *
149  * \subsection jkqtplotter_base_plotsize Plot Sizes & Borders
150  *
151  * JKQTBasePlotter itself is no QWidget. It is merely tool to represent, manage and draw a graph onto any durface.
152  * JKQTPlotter then uses an internal JKQTBasePlotter instance to provide a QWidget.
153  *
154  * Still the JKQTBasePlotter needs a way to represent the location of the graph, the graph axes etc. inside
155  * a drawing rectangle (of size widgetWidth * widgetHeight ). The plot borders are then offsets to the plot
156  * indise that rectangele. This image illustrated the principle:
157  *
158  * \image html plot_widget_orientation.png
159  *
160  * When the plot is actually drawn, it may happen, that additional elements need to be positioned between the
161  * actual coordinate rectangle and the widget worder (e.g. axis labels, ticks, color bars etc.). Therefore
162  * this objects manages a set of internally calculated variables that also incorporate those dynamic elements.
163  * These are e.g. internalPlotBorderLeft, internalPlotBorderTop, internalPlotBorderRight, ... Their names all
164  * start with \c internal... .
165  *
166  *
167  * \subsection jkqtplotter_base_aspectratios Aspect Ratios
168  * First note that this functionality is only available and activated if both axes are linear!
169  *
170  * You can set two different aspect ratios:
171  * - The ratio of plotWidth/plotHeight (setAspectRatio(), setMaintainAspectRatio()) will keep the plots pixel-width and height at a certain value.
172  * - The ratio of (xmax-xmin)/(ymax-ymin) (setAxisAspectRatio(), setMaintainAxisAspectRatio()) will keep the displayed axis ranges in a certain ratio.
173  * .
174  * So to achieve different effects, use these combinations:
175  * - you have a 200x100 range where each 1x1-pixel should have an aspect ratio of 4:
176  * \code
177  * setAspectRatio(200.0/100.0);
178  * setAxisAspectRatio(4.0*getAspectRatio());
179  * \endcode
180  * .
181  *
182  *
183  * \subsection jkqtplotter_base_key Graph Keys (Legends)
184  * This class provides means to plot a key together with the functions. The plotting is only partially implemented in this base class, and has to be
185  * implemented by child classes, as no graph management, that could provide a title for a specific graph, is implemented here. Key Plotting is
186  * performed in the plotKey() method. This method basically draws a frame and background and then calls plotKeyContent() to draw the actual contents.
187  * The size of the key is determined by a call to getKeyExtent(), while it's position and plotting style is determined by the \c key_... parameters.
188  * If getKeyExtent() returns 0 for height or width, no key is plotted. The \c key_... parameters are designed to be rather common for any usual kind
189  * of key, so any implementation of key should be able to use them to parametrize the output. Note however that additional parameters might be defined
190  * in child classes.
191  *
192  * \image html plot_key.png
193  *
194  * As it may be convenient to position the key completely outside the graph this class may change the plotBorders to make the key fit outside
195  * SO if you use a keyPosition \b outside the graph you should NOT keep any space for the key in the margins, as the program will change them to
196  * accomodate the size of the key. If you select an \b inside key position the key will be plotted OVER the graph, i.e. the margins won't be changed
197  * Note that the margin change is internal and not visible in the class interface!
198  *
199  * There is also a possibility to determine the size of the key automatically, so all text fits in. This is activted by the property keyAutosize ( \copybrief keyAutosize ). If this
200  * is \c true the function getKeyExtent() has to check the width of every key item and take it into account when calculating the width and height of the
201  * key content. By default this feature is switched ON.
202  *
203  *
204  * \section jkqtbaseplotter_dataexport_print Printing, Saving & Exporting
205  *
206  * \subsection jkqtplotter_base_saveprint Saving and Printing Graphs
207  * This class implement a set of function to save and print the graphs:
208  * - print() prints the graph on a QPrinter object
209  * - saveAsPixelImage() saves the plot into a pixel image file (PNG, TIFF, ... formats, as supported by Qt)
210  * - saveAsPDF() saves the graph as a PDF file (using the Qt printing engine)
211  * - saveAsPS() saves the graph as a PDF file (using the Qt printing engine)
212  * - saveAsSVG() saves the graph as a SVG file (using the Qt SVG library)
213  * - saveImage() saves the graph
214  * .
215  * You can also copy the contents of the plot into the clipboard:
216  * - copyPixelImage()
217  * .
218  *
219  * \subsection jkqtplotter_base_dataexport Exporting Graph Data
220  * This class implement a set of function to save the data of the graphs:
221  * - saveData() saves the data of the plot
222  * - saveAsCSV() saves the data of the plot as comma separated values
223  * - saveAsSYLK() saves the data of the plot as SYLK spreadsheet
224  * - saveAsDIF() saves the data of the plot as data interchange format file
225  * - saveAsMatlab() saves the data of the plot as a CSV file suitable for Matlab
226  * - saveAsSemicolonSV() saves the data of the plot as a Semicolon Separated Values (SSV)
227  * - saveAsTabSV() saves the data of the plot as a Tabulator Separated Values (CSV) file
228  * - saveAsGerExcelCSV() saves the data of the plot as a Text file (SSV) suitable for german excel, i.e. with comma as decimal separator
229  * .
230  * You can also copy the graphs' data into the clipboard:
231  * - copyData()
232  * - copyDataMatlab()
233  * .

236  * \section JKQTBASEPLOTTER_SYNCMULTIPLOT Synchronizing Several Plots
237  *
238  * Often a single plot is not sufficient, but several plots need to be aligned with respect to each other:
239  *
240  * \image html test_multiplot.png
241  *
242  * This can be achieved by putting several JKQTPlotter instances into a
243  * <a href="http://doc.qt.io/qt-5/layout.html">Qt Layout</a>. Then you can fill each plot differently and
244  * set the x-/y-range of each plot by hand. This method works for simple cases, but has several drawbacks:
245  * - Due to the independent and automatic layouting of each plot, the axes do not need to be aligned properly<br>
246  * \image html jkqtbaseplotter_synchronization_unequalaxes.png
247  * - When you print the plot, the printing does not know about the layout and the other plots in it. Therefor
248  * it will only print the plot itself.<br>
249  * \image html jkqtbaseplotter_synchronization_nogridprint.png "Printing with grid-printing-mode deactivated"<br>
250  * \image html jkqtbaseplotter_synchronization_withgridprint.png "Printing with grid-printing-mode activated"
251  * - when you zoom/pan in one of the plots (e.g. using the mouse), the other plots will not adapt their
252  * axes to match the new area, but especially in cases as in the image above it would be beneficial,
253  * that tha x-axis of the plot at the bottom follows the x-axis of the plot above etc.<br>
254  * \image html jkqtbaseplotter_synchronization_nonsyncedxrange.png
255  * .
256  *
257  *
258  * To overcome these limitations, JKQTPlotter (and JKQTBasePlotter) offer an API with which you can declare relations between
259  * different plots (one of them is made the master) and you can synchronize the axes of two plots, when
260  * zooming (also when calling e.g. zoomToFit() or setXY() ). This API is:
261  * - \ref JKQTBASEPLOTTER_SYNCMULTIPLOT_SYNC
262  * - \ref JKQTBASEPLOTTER_SYNCMULTIPLOT_GRIDPRINT
263  * .
264  *
265  * \subsection JKQTBASEPLOTTER_SYNCMULTIPLOT_SYNC Synchronizing Axis Range & Plot Width
266  *
267  * You can synchronize the plot width and axis range between two JKQTPlotter instaces using:
268  * - synchronizeToMaster() / JKQTBasePlotter::synchronizeToMaster() synchronizes the parent JKQTPlotter with another JKQTPlotter. With two boolean-parameters
269  * you can specify the axes to be synchronized. E.g. in the case above, you would call:
270  * \code
271  * // synchronize width/x-axis of plotResid to width/x-axis of plotMain
272  * plotResid->synchronizeToMaster(plotMain, JKQTBasePlotter::sdXAxis, true, true, true);
273  *
274  * // synchronize y-axis of width/plotResidHist to y-axis of width/plotResid
275  * plotResidHist->synchronizeToMaster(plotResid, JKQTBasePlotter::sdYAxis, true, true, true);
276  * \endcode
277  * This will synchronize the x-axes of the top (\c plotMain ) and bottom-left plot (\c plotResid ),
278  * as well as the y-axes of the bottom-left (\c plotResid ) and bottom-right plot (\c plotResidHist ).
279  * After this call they will have the same size in screen pixels and always span the same range
280  * in plot coordinates.
281  * - synchronizeXToMaster() / JKQTBasePlotter::synchronizeXToMaster() like synchronizeToMaster() / JKQTBasePlotter::synchronizeToMaster(),
282  * but synchronizes only the x-axes
283  * - synchronizeYToMaster() / JKQTBasePlotter::synchronizeYToMaster() like synchronizeToMaster() / JKQTBasePlotter::synchronizeToMaster(),
284  * but synchronizes only the y-axes
285  * - resetMasterSynchronization() / JKQTBasePlotter::resetMasterSynchronization() deletes all synchronizations
286  * from the JKQTPlotter
287  * .
288  *
289  * \see See \ref JKQTPlotterMultiPlotLayout for an extensive example of the functionality.
290  *
291  *
292  * \subsection JKQTBASEPLOTTER_SYNCMULTIPLOT_GRIDPRINT Grid Printing/Desclaring Relations between Plots
293  *
294  * Usually each JKQTPlotter/JKQTBasePlotter prints or exports only itself. But in many cases you might want to
295  * add several plots that form a grid layout and also want to export them as one image/print them on one page.
296  * To do this, JKQTPlotter/JKQTBasePlotter offers the <b>"grid printing" mode</b>. In this mode, you can declare
297  * relations between different JKQTPlotters/JKQTBasePlotters by putting them in a rectangular grid (like in a
298  * <a reh="http://doc.qt.io/qt-5/qgridlayout.html">QGridLayout</a>). then any export/print action will draw all (explicitly)
299  * decalred graphs.
300  *
301  * \image html jkqtplotter_gridprint.png
302  *
303  * The grid printing API is composed of these functions:
304  * - setGridPrinting() enables grid printing for this JKQTPlotter. If set to \c true , and you print afterwards,
305  * the printout (or export) will not only contain the plot itself, but also additional plots that were
306  * declared using addGridPrintingPlotter() (see below).
307  * - addGridPrintingPlotter() add a new plotter \a plotterOther for grid printing mode, at location \a x / \a y
308  * E.g. in the example shown above, you could call:
309  * \code
310  * plotMain->setGridPrinting(true);
311  * plotMain->addGridPrintingPlotter(0,1,plotResid);
312  * plotMain->addGridPrintingPlotter(1,1,plotResidHist);
313  * \endcode
314  * - setGridPrintingCurrentX() / setGridPrintingCurrentY() / setGridPrintingCurrentPos()
315  * sets the location of the calling plot inside the grid. <i>By default each plot assumes to be at (0,0).</i>
316  * - clearGridPrintingPlotters() clear all additional plotters for grid printing mode
317  * .
318  *
319  * \see See \ref JKQTPlotterMultiPlotLayout for an extensive example of the functionality.
320  *
