57 /*! \brief a speed optimized plotter class
58  \ingroup jkqtfastplotter
59 
60  This plotter class is (in contrast to JKQTPlotter) optimized for speed. It allows to plot function graphs/data in
61  a simple manner. The focus of design was not on a nice output (athough this is not unimportant), but on the speed
62  of the output. So the class may be used to di quick plotting of data, i.e. in fast image sequence applications.
63 
64  These measures were taken to increase speed:
65  - The coordinate system is drawn onto an internal image which is only updated, when the size of the widget
66  or a property of the coordinate system changes. When only the data changes, just the data plot has to be
67  updated.
68  - The class uses an internal buffer with the complete image, so a repaint does not always trigger a complete
69  repaint of all elements, but just measns painting the internal image to the screen. This also allows to
70  plot selections ... over the plot, without having to replot the plots.
71  - The definition of the coordinate system as well as the definition the system ticks ... is reduced to the absolute
72  minimum.
73  - The method setPlotUpdateEnabled() allows to prevent replotting of the class contents, e.g. when you want to set
74  a multitude of parameters without triggering a replot every time.
75  .
76 
77  The class supports these features:
78  - coordinate axes may be linear or logarithmic.
79  - ticks and tick labels are drawn by a simple algorithm: The user has to supply two values:
80  \c zeroTick is a tick value that exists in any case (i.e. 0) and \c tickDistance which contains the distance
81  between two ticks (only applicable to linear axes). Then the class starts to draw at zeroTick and move on to
82  \c zeroTick+tickDistance and \c zeroTick-tickDistance until \c min / \c max of the axis is reached. For logarithmic axes
83  the class uses \c zeroTick*10 and \c zeroTick/10
84  - plot may contain a grid (x and y grid may be switched on/off separately).
85  - plots are represented by descendents of JKQTFPPlot and the plotting code is completely
86  contained therein (and should be speed optimized).
87  - it is possible to keep a specified aspect ratio
88  - it is possible to couple different plots in a way, that they are plottetd with the same width/height
89  both in pixel and world coordinates by using plotterSizesChanged() and synchronizeX() / synchronizeY().
90  .
91 
92 */

404  /** \brief return x-pixel coordinate from x coordinate */
405  inline double x2p(double x) {
406  if (xAxisLog) {
407  if (x<0) return xOffset+log(xMin/10.0)/log(10.0)*xScale;
408  return xOffset+log(x)/log(10.0)*xScale;
409  } else {
410  return xOffset+x*xScale;
411  }
412  }

414  /** \brief return x coordinate coordinate from x-pixel */
415  inline double p2x(long x) {
416  if (xAxisLog) {
417  return exp(log(10.0)*(static_cast<double>(x)-xOffset)/(xScale));
418  } else {
419  return (static_cast<double>(x)-xOffset)/(xScale);
420  }
421  }
422 
423  /** \brief return y-pixel coordinate from y coordinate */
424  inline double y2p(double y) {
425  if (yAxisLog) {
426  if (y<0) return yOffset-log(yMin/10.0)/log(10.0)*yScale;
427  return yOffset-log(y)/log(10.0)*yScale;
428  } else {
429  return yOffset-y*yScale;
430  }
431  }
432 
433  /** \brief return y coordinate coordinate from y-pixel */
434  inline double p2y(long y) {
435  if (yAxisLog) {
436  return exp(log(10.0)*(static_cast<double>(y)-yOffset)/(-1.0*yScale));
437  } else {
438  return (static_cast<double>(y)-yOffset)/(-1.0*yScale);
439  }
440  }
