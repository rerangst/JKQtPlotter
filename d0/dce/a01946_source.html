52 /*! \brief this class parses a LaTeX string and can then draw the contained text/equation onto a <a href="http://doc.qt.io/qt-5/qpainter.html">QPainter</a>
53  \ingroup jkqtmathtext
54 
55 
56  JKQTMathText is a self-contained LaTeX-renderer for Qt. It is used to renderer
57  labels in JKQTPlotter/JKQTBasePlotter, but can be used independently.
58  The class does not depend on any library, except Qt.
59  In particular it actually parses a LaTeX string and draws it in pure C++. It does NOT rely
60  on an installed LaTeX for the rendering!
61 
62  \section JKQTMathTextUsage Usage
63  \subsection JKQTMathTextUsageDirect Direct Usage
64  This small piece of C++ code may serve as an example of the usage and capabilities of the class:
65  \code
66  // create a JKQTMathText object.
67  JKQTMathText mathText;
68 
69  // configure its properties to influence the rendering (e.g. fonts to use, font size, ...)
70  mathText.useXITS();
71  mathText.setFontSize(20);
72 
73  // parse some LaTeX code (the Schroedinger's equation)
74  mathText.parse("$\\left[-\\frac{\\hbar^2}{2m}\\frac{\\partial^2}{\\partial x^2}+V(x)\\right]\\Psi(x)=\\mathrm{i}\\hbar\\frac{\\partial}{\\partial t}\\Psi(x)$");
75 
76  // use the draw() methods to draw the equation using a QPainter (here onto a QPixmap)
77  QPainter painter;
78  QPixmap pix(600,400);
79  painter.begin(&pix);
80  mathText.draw(painter, Qt::AlignCenter, QRectF(0,0,pix.width(), pix.height()), false);
81  painter.end();
82  \endcode

92  \subsection JKQTMathTextErrorHandling Error Handling
93 
94  The class is designed to be as robust as possible and will still return some output, even if the equation contains some errors.
95  Nevertheless, several errors are detected while parsing. You can get a list of error messages using getErrorList() after calling parse().
96  Also parse() will return \c false if an error occured while parsing.

113  \section JKQTMathTextSuppoertedLaTeX Supported LaTeX Subset
114  \subsection JKQTMathTextSuppoertedLaTeXSimple Simple Instructions
115  The supported LaTeX subset is listes below. Please note that some commands are used differently than in actual LaTeX.
116  For example \c \\bf is just a renamed form of \c \\textbf and used as \c \\bf{...} and NOT as \c {\\bf...} .
117 
118  - \c $ : enter/leave math mode
119  - \c \\bf{...} \c \\textbf{...} \c \\mathbf{...}: draw the contained text in bold face \image html jkqtmathtext_bold.png
120  - \c \\it{...} \c \\textit{...} \c \\mathit{...} : draw the contained text in italic face \image html jkqtmathtext_italic.png
121  - \c \\rm{...} \c \\textrm{...} \c \\mathrm{...} \c \\mbox{...} : draw the contained text in normal upright roman font face \image html jkqtmathtext_fonts.png
122  - \c \\sf{...} \c \\textsf{...} \c \\mathsf{...} : draw the contained text in normal upright sans-serif font face \image html jkqtmathtext_fonts.png
123  - \c \\script{...} \c \\textscript{...} \c \\mathscript{...} : draw the contained text in a script font face \image html jkqtmathtext_fonts.png
124  - \c \\sc{...} : draw the text in small caps \image html jkqtmathtext_sc.png
125  - \c \\ul{...} \c \\underline{...} \c \\underlined{...} : draw the text with underlining \image html jkqtmathtext_ul.png
126  - \c \\ol{...} \c \\overline{...} \c \\overlined{...} : draw the text with overlining \image html jkqtmathtext_ol.png
127  - \c \\tt{...} \c \\texttt{...} \c \\mathtt{...} : draw text in typewriter font \image html jkqtmathtext_fonts.png
128  - \c \\textcolor{color}{...} \c \\color{color} \c \\mathcolor{color}{...} : draw colored text \image html jkqtmathtext_colored.png
129  - \c \\boxed{...} : draw text with a box around it \image html jkqtmathtext_boxed.png
130  - \c \\colorbox{color}{...} : draw a colored box around text \image html jkqtmathtext_colorbox.png
131  - \c \\alpha ... : display the according greek letter \image html jkqtmathtext_greek.png
132  - \c ^{...} \c _{...} : display the contents of braces in superscript/subscript \image html jkqtmathtext_supersub.png
133  <br>Special subscript/superscript typesetting applies, when the sub/super follows \c \\sum \c \\Prod ...: \image html jkqtmathtext_specialsubsuper.png
134  - \c \\{ / \\} : display opening/closing brace
135  - \c \\_ : display underscore
136  - \c \\sum \c \\prod \c \\int ... : plot special symbol. Note that depending on the fontEncoding the available
137  symbols may differ (there are not all symbols defined in the MS Windows Symbol
138  font!). Best coverage should be given by Unicode font encoding with a good
139  unicode font installed!<br>\image html jkqtmathtext_symbols.png
140  - <code>\\vec{x} \\dot{x} \\ddot{x} \\overline{x} \\underline{x} \\hat{x} \\tilde{x} \\uul{x} \\ool{x} \\bar{x} \\arrow{x} </code>: Decorations over/under symbols \image html jkqtmathtext_mathdeco.png
141  - <code>\\verb{don't parse this _aaa\LaTeX} </code>: interpret enclosed text as verbose \image html jkqtmathtext_verb.png
142  .

144  \subsection JKQTMathTextSuppoertedLaTeXBraces Braces ...
145  Braces in math mode are adjusted in size, so they are a small bit (factor \c brace_factor ) larger than the contents.
146  To enable this you have to write braces with \c \\left and \c \\right. These types of braces are defined (slight
147  differences to LaTeX standard):
148  - \c \\left( \c \\right) : default meaning (), \image html jkqtmathtext_brace_round.png
149  - \c \\left[ \c \\right] : default meaning [], \image html jkqtmathtext_brace_rect.png
150  - \c \\left\\{ \c \\right\\} : default meaning {}, \image html jkqtmathtext_brace_curly.png
151  - \c \\left< \c \\right> : "averaging" braces , \image html jkqtmathtext_brace_tri.png
152  - \c \c \\left\\lfloor \c \\right\\rfloor : floor braces , \image html jkqtmathtext_brace_floor.png
153  - \c \\left~ \c \\right~ \c \\left\\lceil \c \\right\\rceil : ceil braces , \image html jkqtmathtext_brace_ceil.png
154  - \c \\left| \c \\right| : absolute value braces | |, \image html jkqtmathtext_brace_oneline.png
155  - \c \\left\| \c \\right\| : norm braces || ||, \image html jkqtmathtext_brace_dblline.png
156  - You can use \c \\left. or \c \\right. to have only right or only left brace
157  .

159  \subsection JKQTMathTextSuppoertedLaTeXUnderOver Undersetting, Oversetting, Underbraces, Overbraces ...
160  There are also instructions that allow to under/overset braces, arrows, ...:
161  - <code>$\underbrace{x+x+...+x}{k\ \mathrm{times}}$</code> \image html jkqtmathtext_brace_underbrace.png
162  - <code>$\overbrace{x+x+...+x}{k\ \mathrm{times}}$</code> \image html jkqtmathtext_brace_overbrace.png
163  - <code>$\overset{main}{over}$</code> \image html jkqtmathtext_brace_overset.png
164  - <code>$\underset{main}{under}$</code> \image html jkqtmathtext_brace_underset.png

180  \section JKQTMathTextSuppoertedFonts Font Handling
181 
182  Several fonts are defined as properties to the class:
183  - A "roman" font used as the standard font ( setFontRoman() )
184  - A "sans-serif" font which may be activated with \c \\sf ... ( setFontSans() )
185  - A "typewriter" font which may be activated with \c \\tt ... ( setFontTypewriter() )
186  - A "script" font which may be activated with \c \\script ... ( setFontScript() )
187  - A greek font which is used to display greek letters \c \\alpha ... ( setFontGreek() )
188  - A symbol font used to display special (math) symbols.
189  - A "roman" font used as the standard font in math mode
190  - A "sans-serif" used as sans serif font in math mode
191  - A "blackboard" font used to display double stroked characters (setFontBlackboard() )
192  - A "caligraphic" font used to display caligraphic characters ( setFontCaligraphic() )
193  .
194 
195  These fonts are generic font classes, which font is actually used can be configured in JKQTMathText class with the \c set...() functions mentioned above. You can also use these functions to set the fonts used for math rendering in math-mode:
196  - useSTIX() use the STIX fonts from <a href="https://www.stixfonts.org/">https://www.stixfonts.org/</a> in math-mode<br>\image html jkqtmathparser_stix.png
197  - useXITS() use the XITS fonts from <a href="https://github.com/alif-type/xits">https://github.com/alif-type/xits</a> in math-mode. These are included by default in this library and also activated by default.<br>\image html jkqtmathparser_xits.png
198  - useASANA() use the ASANA fonts from <a href="https://ctan.org/tex-archive/fonts/Asana-Math/">https://ctan.org/tex-archive/fonts/Asana-Math/</a> in math-mode<br>\image html jkqtmathparser_asana.png
199  - useAnyUnicode() use generic Unicode fonts, e.g. "Arial" and "Times New Roman" in math-mode. You should use fonts that contain as many of the mathematical symbols as possible to ensure good rendering results.<br>using "Times New Roman": \image html jkqtmathparser_timesnewroman.png
200  <br>using "Arial": \image html jkqtmathparser_arial.png
201  <br>using "Courier New": \image html jkqtmathparser_couriernew.png
202  <br>using "Comic Sans MS": \image html jkqtmathparser_comicsans.png
203  .
204 
205 
206  Math-mode is activated by enclosing your equation in \c $...$ or \c \\[...\\] . This mode is optimized for mathematical equations. Here is an example of the difference:
207  - <b>math-mode (XITS fonts are used, whitespaces are mostly not drawn directly, symbol spacing is different)</b> \c $...$: <br>\image html jkqtmathparser_schreq_mathmode.png
208  - <b>normal mode (Times new Roman is used, whitespaces are evaluated directly)</b>: <br>\image html jkqtmathparser_schreq_normalmode.png
